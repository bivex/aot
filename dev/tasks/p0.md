# Task P0: English Subject-Predicate Detection

## Status: Implemented (Morphology + Grammar + Heuristics)

The English syntactic module now has a complete pipeline: UniMorph-based morphology with closed-class verb paradigms, a 61-rule GLR grammar, and heuristic SP detection.

### Changes Implemented:

1.  **Morphology (UniMorph + Closed Classes)**:
    - Integrated UniMorph English data (~540K lemmas) via `unimorph_to_aot_v2.py`.
    - Injected closed-class suppletive verb paradigms: **BE** (is/am/are/was/were/been/being), **HAVE** (have/has/had/having), **DO** (do/does/did/done/doing) â€” tagged as VBE.
    - Modal verbs (can/must/will/shall/may/ought) tagged as MOD.
    - Particles (not, never) injected as PART, since UniMorph mis-tags them as ADV.
    - 17 POS tags: NOUN, ADJ, VERB, VBE, MOD, NUMERAL, CONJ, INTERJ, PREP, PARTICLE, ARTICLE, ADV, PN, ORDNUM, PRON, POSS, PN_ADJ.

2.  **Grammar (`simple_syn.grm` â€” 61 rules)**:
    - NP: bare nouns, pronouns, proper nouns; with articles, adjectives, possessives, numerals.
    - VP: simple verbs, copula (VBE+ADJ/NP/PP), progressive (is running), passive (was broken), perfect (has eaten), perfect progressive (has been running), passive progressive (is being eaten).
    - Modal: can/must/will + verb, + progressive, + passive.
    - Negation: VBE/VOD + not + verb/adj/ing/pp.
    - Infinitive with "to": is to go.
    - Adverbial modifiers in VP.
    - Coordination: NP and NP, VP and VP, verb and verb.
    - Subordinate clauses: SP + CONJ + SP, CONJ + NP + VP.
    - Root: SYN_GROUP covers NP, VP, PP, ADVERB, PART, SP.

3.  **Subject-Predicate Heuristics** (`CEngSentence::BuildSubjAndPredRelation`):
    - Locates the first verb-like unit as **Predicate**, closest preceding noun-like unit as **Subject**.
    - Assigns `isSubj: true` and `m_iPredk` for visualization.

4.  **Stability Fixes**:
    - Fixed off-by-one in `CoordConjTypesCount` (21â†’20).
    - Separated coordinating vs subordinating conjunctions in `EngOborDic`.
    - GLR chunker bypassed for English (requires `BuildTerminalSymbolsByWord` adaptation).

### Current Limitations:

- **GLR integration**: Grammar items from `.grm` don't match morphological homonym symbols â€” `BuildTerminalSymbolsByWord` needs adaptation so the GLR parser can consume VBE/MOD/PART tags.
- **Stub methods**: `GleicheSubjectPredicate`, `is_infinitive`, `is_morph_participle`, etc. in `CEngGramTab` always return false.
- **SP disambiguation**: Heuristic picks the first verb, which is wrong for words like CAT (both NOUN and VERB). Should prefer VBE/MOD or verb with NOUN homonym on preceding position.
- **Frontend localization**: Legend labels in synan.js are localized per language, but the syntax tree visualization is language-agnostic.
