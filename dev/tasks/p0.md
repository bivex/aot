# Task P0: English Subject-Predicate Detection

## Status: Implemented (Heuristic Prototype)

The English syntactic module (`EngSynanLib`) was previously a minimal prototype that only performed basic phrasal chunking (NP, VP, PP) without identifying deep syntactic relations like Subject-Predicate (SP).

### Changes Implemented:

1.  **Subject-Predicate Heuristics**: 
    Implemented `CEngSentence::BuildSubjAndPredRelation` with a search heuristic:
    - Locates the first verb-like unit in a clause as the **Predicate**.
    - Locates the closest preceding noun-like unit as the **Subject**.
    - Assigns `isSubj: true` and `m_iPredk` to enable visual representation in the syntax tree.

2.  **Morphology Workarounds**:
    Since the current English morphological module is incomplete (often labeling verbs as NOUN), I added a temporary POS hardcoding in `InitHomonymMorphInfo` for common English verbs (`is`, `was`, `sat`, `read`, etc.) to enable the syntax heuristics to function.

3.  **Grammar Updates**:
    Updated `simple_syn.grm` to include an `[SP]` rule that combines `[NP]` and `[VP]`, allowing the parser to recognize basic sentence structures as a single group.

4.  **Stability Fixes**:
    - Bypassed the GLR chunker for English to prevent crashes on uninitialized data structures.
    - Fixed `SyntaxInit` to correctly handle missing English `oborot` dictionaries.
    - Implemented missing POS check methods in `CEngGramTab`.

### Current Limitations:

- **Morphology**: Relies on a hardcoded list of verbs. For a production-ready system, the English UniMorph data (added as submodule) should be properly integrated into the binary morphology builder.
- **Complexity**: Heuristics are currently limited to simple "Subject Verb" patterns and do not yet handle complex coordination, passive voice, or nested relative clauses.

## Next Steps:
- Integrate UniMorph dictionary into the English morphological builder.
- Re-enable GLR parsing once English-specific rules and precompiled tables are properly generated.
