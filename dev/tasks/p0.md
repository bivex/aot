# P0: English SynanDaemon hangs on sentences longer than 3 words

## Symptom

The SynanDaemon enters an infinite loop when processing English sentences longer than 3 words. The daemon is single-threaded, so a single hung request blocks all subsequent requests (including those in other languages).

- "I am." works
- "The cat sat on the mat." hangs forever
- "I am." hangs if preceded by a hung request

## Reproduction

```bash
RML=$(pwd) ./Bin/SynanDaemon --host 127.0.0.1 --port 8089
curl --max-time 10 "http://127.0.0.1:8089/?dummy=1&action=syntax&langua=English&query=The%20cat%20sat%20on%20the%20mat."
# hangs
```

## Root cause

There is an infinite loop in `RunSyntaxInClauses` for English. Logs show `CClause::BuildGroups` being called repeatedly for the same clause in a cycle that never converges:

```
RunSyntaxInClauses: clauses count = 1
RunSyntaxInClauses: processing clause 0
CClause::BuildGroups: starting for clause (0, 2)
BuildSubjAndPredRelation: ClauseType=65535 RootWordNo=-1 lang=1
...
CClause::BuildGroups: after trigger
RunSyntaxInClauses: clauses count = 1    <-- repeats forever
RunSyntaxInClauses: processing clause 0
```

`ClauseType=65535` (= `-1` as `size_t`) and `RootWordNo=-1` indicate that the clause type is never properly assigned for English, so the loop never reaches a stable state.

## Likely fix location

- `Source/synan/SynCommonLib/Sentence.cpp` — `RunSyntaxInClauses` needs a maximum iteration guard or convergence check
- English clause type assignment in `Source/synan/EngSynan/` — investigate why `InitClauseType` produces no valid type

## Pre-existing

This is a pre-existing bug in the original codebase. It is NOT caused by the Ukrainian AnalyticVerb commit (`d1aa21bb`). This was verified by testing the parent commit (`1c03c576`) — the same hang occurs.
